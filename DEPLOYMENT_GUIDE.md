# 🚀 Coolify Deployment Guide

**Complete guide for deploying the unified fullstack workload management system to Coolify**

## 📋 Prerequisites

- ✅ VPS with Coolify installed
- ✅ Domain name configured 
- ✅ GitHub repository with the fullstack code
- ✅ Current Supabase credentials for migration

---

## 🏗️ Coolify Project Setup

### 1. Create New Project

1. **Login to Coolify Dashboard**
2. **Create New Project**
   - Name: `workload-management`
   - Description: `Unified fullstack workload management system`

### 2. Add GitHub Repository

1. **Go to Sources** → **Add New Source**
2. **Connect GitHub Repository**
   - Repository: `your-org/workload-management-fullstack`
   - Branch: `main`
   - Build Pack: `Docker`

### 3. Configure Services

#### PostgreSQL Database

1. **Add New Resource** → **Database** → **PostgreSQL**
   - Name: `workload-postgres`
   - Version: `15`
   - Database Name: `workload_production`
   - Username: `workload_user`
   - Password: `<generate-secure-password>`

#### Redis Cache (Optional)

1. **Add New Resource** → **Database** → **Redis**
   - Name: `workload-redis` 
   - Version: `7`

#### Main Application

1. **Add New Resource** → **Application**
   - Name: `workload-app`
   - Source: Select your GitHub repository
   - Build Pack: `Docker`
   - Dockerfile: `docker/Dockerfile`
   - Port: `3000`

## ⚙️ Environment Variables Configuration

### Required Environment Variables

Set these in **Coolify Dashboard** → **Your App** → **Environment Variables**:

```bash
# Database (auto-generated by Coolify)
DATABASE_URL=postgresql://workload_user:password@workload-postgres:5432/workload_production

# JWT Authentication (generate secure keys)
JWT_SECRET=<generate-32-char-random-string>
JWT_REFRESH_SECRET=<generate-32-char-random-string>

# Application Configuration
NODE_ENV=production
ALLOWED_ORIGINS=https://yourdomain.com
UPLOAD_PATH=/app/uploads

# Optional Redis
REDIS_URL=redis://workload-redis:6379

# Migration (temporary - for one-time migration)
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key
```

### Generate Secure Keys

```bash
# JWT Secret (32 characters)
openssl rand -hex 32

# JWT Refresh Secret (32 characters)
openssl rand -hex 32
```

## 🔄 Migration Process

### Phase 1: Pre-Migration Setup

1. **Verify Services are Running**
   ```bash
   # Check PostgreSQL
   curl -f https://yourdomain.com/api/health/db || echo "Database not ready"
   
   # Check Application
   curl -f https://yourdomain.com/api/health || echo "App not ready"
   ```

2. **Access the Application**
   - Navigate to `https://yourdomain.com`
   - Should show the login page (without data)

### Phase 2: Data Migration

1. **SSH into the Coolify Server**
   ```bash
   ssh your-vps-user@your-vps-ip
   ```

2. **Access the Application Container**
   ```bash
   # Find the container
   docker ps | grep workload-app
   
   # Access container
   docker exec -it <container-id> bash
   ```

3. **Run Database Migration**
   ```bash
   # Inside the container
   cd /app
   
   # Test migration first
   npm run migrate:supabase -- --dry-run
   
   # Run actual migration
   npm run migrate:supabase
   ```

4. **Verify Migration**
   ```bash
   # Check data in database
   npx prisma studio --browser none --port 5555
   
   # Or use database queries
   npx prisma db seed
   ```

### Phase 3: Post-Migration Verification

1. **Test Application Features**
   - ✅ User login with migrated credentials
   - ✅ Dashboard statistics display correctly
   - ✅ Workload data shows up properly
   - ✅ Calendar events are accessible
   - ✅ Real-time updates work
   - ✅ File uploads function

2. **Performance Testing**
   ```bash
   # Test API endpoints
   curl -X POST https://yourdomain.com/api/auth/login \
        -H "Content-Type: application/json" \
        -d '{"username":"admin","password":"admin123"}'
   
   # Test dashboard
   curl https://yourdomain.com/api/dashboard/stats
   ```

## 🔧 Coolify-Specific Configuration

### Docker Labels for Coolify

Add these labels to your `docker-compose.prod.yml`:

```yaml
services:
  app:
    labels:
      - "coolify.managed=true"
      - "coolify.version=latest"
      - "coolify.type=application"
      - "traefik.enable=true"
      - "traefik.http.routers.workload-app.rule=Host(`yourdomain.com`)"
      - "traefik.http.routers.workload-app.tls.certresolver=letsencrypt"
      - "traefik.http.services.workload-app.loadbalancer.server.port=3000"
```

### Health Checks

Create health check endpoint:

```typescript
// app/api/health/route.ts
import { NextResponse } from 'next/server'
import { checkDatabaseConnection } from '@/lib/database/prisma'

export async function GET() {
  try {
    const dbHealth = await checkDatabaseConnection()
    
    return NextResponse.json({
      status: 'healthy',
      timestamp: new Date().toISOString(),
      database: dbHealth.status,
      uptime: process.uptime()
    })
  } catch (error) {
    return NextResponse.json(
      { status: 'error', error: 'Health check failed' },
      { status: 500 }
    )
  }
}
```

### Automatic Backups

Configure in Coolify:

1. **PostgreSQL Service** → **Backups**
   - Schedule: `Daily at 2:00 AM`
   - Retention: `7 days`
   - S3 Storage: Configure if available

## 🔒 Security Configuration

### SSL/TLS Setup

Coolify automatically handles SSL with Let's Encrypt:

1. **Domain Configuration**
   - Add your domain in Coolify
   - DNS should point to your VPS IP
   - SSL certificate will be auto-generated

### Firewall Rules

```bash
# Allow HTTP/HTTPS
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# Allow SSH
sudo ufw allow 22/tcp

# PostgreSQL (only from localhost)
sudo ufw allow from 127.0.0.1 to any port 5432

# Enable firewall
sudo ufw enable
```

### Security Headers

Add to `next.config.js`:

```javascript
const nextConfig = {
  async headers() {
    return [
      {
        source: '/(.*)',
        headers: [
          {
            key: 'X-Frame-Options',
            value: 'DENY'
          },
          {
            key: 'X-Content-Type-Options',
            value: 'nosniff'
          },
          {
            key: 'Referrer-Policy',
            value: 'origin-when-cross-origin'
          },
          {
            key: 'X-XSS-Protection',
            value: '1; mode=block'
          }
        ]
      }
    ]
  }
}
```

## 📊 Monitoring & Maintenance

### Application Monitoring

1. **Coolify Dashboard** provides:
   - CPU/Memory usage
   - Container logs
   - Service status
   - Deployment history

2. **Custom Monitoring**
   ```typescript
   // lib/monitoring.ts
   export async function logMetrics() {
     const stats = {
       timestamp: new Date(),
       memoryUsage: process.memoryUsage(),
       uptime: process.uptime(),
       activeConnections: getActiveConnections()
     }
     
     console.log('METRICS:', JSON.stringify(stats))
   }
   ```

### Log Management

```bash
# View application logs
docker logs -f <container-id>

# View PostgreSQL logs
docker logs -f <postgres-container-id>

# View system logs
journalctl -u coolify -f
```

### Database Maintenance

```sql

VACUUM ANALYZE;
REINDEX DATABASE workload_production;

-- Check database size
SELECT pg_size_pretty(pg_database_size('workload_production'));

-- Monitor active connections
SELECT count(*) as active_connections FROM pg_stat_activity;
```

## 🚨 Troubleshooting

### Common Deployment Issues

**Build Fails**
```bash
# Check build logs in Coolify
# Common fix: Update package.json dependencies
```

**Database Connection Issues**
```bash
# Verify environment variables
echo $DATABASE_URL

# Check PostgreSQL is running
docker ps | grep postgres

# Test connection
npx prisma db push
```

**Migration Errors**
```bash
# Reset and re-run migration
docker exec -it <container> bash
cd /app
npm run db:reset
npm run migrate:supabase
```

**Real-time Not Working**
```bash
# Check WebSocket endpoint
curl -I -N -H "Connection: Upgrade" \
     -H "Upgrade: websocket" \
     https://yourdomain.com/api/realtime
```

### Performance Issues

**Slow Database Queries**
```sql
-- Enable query logging
ALTER SYSTEM SET log_statement = 'all';
SELECT pg_reload_conf();

-- Check slow queries
SELECT query, calls, total_time, mean_time 
FROM pg_stat_statements 
ORDER BY total_time DESC LIMIT 10;
```

**High Memory Usage**
```bash
# Monitor container resources
docker stats <container-id>

# Optimize Node.js memory
export NODE_OPTIONS="--max-old-space-size=512"
```

### Emergency Procedures

**Rollback Deployment**
1. Go to Coolify Dashboard
2. **Deployments** → **Previous Version**
3. Click **Redeploy**

**Database Recovery**
```bash
# Restore from backup
docker exec -i postgres_container pg_restore -U workload_user -d workload_production < backup.sql

# Or use Coolify backup restore
```
