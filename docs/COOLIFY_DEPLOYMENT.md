# 🚀 Coolify Deployment Guide

Complete step-by-step guide for deploying HPSB Workload Management System to production using Coolify.

## 📋 Prerequisites

### VPS Requirements
- **OS**: Ubuntu 22.04 LTS
- **Memory**: 4GB RAM minimum (8GB recommended)
- **CPU**: 2 vCPU minimum
- **Storage**: 50GB minimum
- **Network**: Public IP with domain name

### Domain Setup
- Domain: `workload.kemlu.go.id`
- Subdomain for staging: `staging.workload.kemlu.go.id`
- DNS A records pointing to your VPS IP

---

## Phase 1: VPS Preparation

### 1.1 Run VPS Setup Script
```bash
# Download and run the automated setup script
curl -fsSL https://raw.githubusercontent.com/USERNAME/hpsb-workload-management/main/scripts/vps-setup.sh | bash

# Or clone the repository and run locally
git clone https://github.com/USERNAME/hpsb-workload-management.git
cd hpsb-workload-management
chmod +x scripts/vps-setup.sh
./scripts/vps-setup.sh
```

### 1.2 Verify Installation
```bash
# Check Docker
docker --version
docker compose version

# Check Coolify
curl -k https://localhost:8080/api/health

# Check firewall
sudo ufw status
```

### 1.3 Access Coolify
1. Open browser: `https://YOUR_VPS_IP:8080`
2. Complete initial setup wizard
3. Create admin account
4. Configure base settings

---

## Phase 2: Coolify Configuration

### 2.1 Create Project
1. Navigate to **Projects** → **New Project**
2. Project Name: `HPSB Workload Management`
3. Description: `Production workload management system for HPSB`

### 2.2 Add GitHub Repository
1. Go to **Sources** → **New Source**
2. Type: **GitHub**
3. Repository: `https://github.com/USERNAME/hpsb-workload-management.git`
4. Branch: `main` (production), `staging` (staging)
5. Add deploy key to GitHub repository

### 2.3 Setup Environments

#### Production Environment
```yaml
Name: production
Domain: workload.kemlu.go.id
SSL: Auto (Let's Encrypt)
Branch: main
Build Pack: Docker
Dockerfile: ./docker/Dockerfile
```

#### Staging Environment
```yaml
Name: staging
Domain: staging.workload.kemlu.go.id
SSL: Auto (Let's Encrypt)
Branch: staging
Build Pack: Docker
Dockerfile: ./docker/Dockerfile
```

## Phase 3: Database Setup

### 3.1 Create PostgreSQL Service

#### Production Database
```yaml
Service Type: PostgreSQL 15
Service Name: workload-postgres-prod
Database Name: workload_production
Username: workload_user
Password: [Generate 32-character password]

Resource Limits:
  Memory: 1GB
  CPU: 1 vCPU
  Storage: 20GB

Configuration:
  shared_buffers: 256MB
  effective_cache_size: 1GB
  maintenance_work_mem: 64MB
```

#### Staging Database
```yaml
Service Type: PostgreSQL 15
Service Name: workload-postgres-staging
Database Name: workload_staging
Username: workload_user_staging
Password: [Generate password]

Resource Limits:
  Memory: 512MB
  CPU: 0.5 vCPU
  Storage: 10GB
```

### 3.2 Create Redis Service

#### Production Redis
```yaml
Service Type: Redis 7
Service Name: workload-redis-prod
Configuration: default
Resource Limits:
  Memory: 512MB

```

#### Staging Redis
```yaml
Service Type: Redis 7
Service Name: workload-redis-staging
Configuration: default

  Memory: 256MB
```

## Phase 4: Environment Variables

### 4.1 Production Environment Variables

```bash
# Application
NODE_ENV=production
PORT=3000
HOSTNAME=0.0.0.0

# Database (auto-generated by Coolify)
DATABASE_URL=postgresql://workload_user:[PASSWORD]@workload-postgres-prod:5432/workload_production

# Redis (auto-generated by Coolify)
REDIS_URL=redis://workload-redis-prod:6379

# Security (Generate with: openssl rand -hex 32)
JWT_SECRET=your_32_character_jwt_secret_here
JWT_REFRESH_SECRET=your_32_character_refresh_secret_here

# Application Configuration
ALLOWED_ORIGINS=https://workload.kemlu.go.id
UPLOAD_PATH=/app/uploads

# Features
NEXT_TELEMETRY_DISABLED=1
LOG_LEVEL=info
ENABLE_METRICS=true
HEALTH_CHECK_TIMEOUT=5000

# Backup
BACKUP_RETENTION_DAYS=30
BACKUP_COMPRESSION=true
```

### 4.2 Staging Environment Variables

```bash
# Application
NODE_ENV=staging
PORT=3000
HOSTNAME=0.0.0.0

# Database
DATABASE_URL=postgresql://workload_user_staging:[PASSWORD]@workload-postgres-staging:5432/workload_staging

# Redis
REDIS_URL=redis://workload-redis-staging:6379

# Security (Different from production)
JWT_SECRET=your_staging_jwt_secret_here
JWT_REFRESH_SECRET=your_staging_refresh_secret_here

# Application Configuration
ALLOWED_ORIGINS=https://staging.workload.kemlu.go.id
UPLOAD_PATH=/app/uploads

# Features
NEXT_TELEMETRY_DISABLED=1
LOG_LEVEL=debug
ENABLE_METRICS=true
```

## Phase 5: Deployment Configuration

### 5.1 Production Deployment
1. **Build Settings**
   - Build Command: `npm run build`
   - Build Context: `/`
   - Dockerfile Path: `./docker/Dockerfile`

2. **Deploy Settings**
   - Auto Deploy: Disabled (manual approval)
   - Health Check Path: `/api/health`
   - Health Check Interval: 30s
   - Restart Policy: `unless-stopped`

3. **Resource Limits**
   - Memory: 1GB limit, 512MB reservation
   - CPU: 1 vCPU limit, 0.5 vCPU reservation

### 5.2 Staging Deployment
1. **Build Settings**
   - Build Command: `npm run build`
   - Auto Deploy: Enabled on push to `staging` branch

2. **Resource Limits**
   - Memory: 512MB limit, 256MB reservation
   - CPU: 0.5 vCPU limit, 0.25 vCPU reservation

## Phase 7: Monitoring & Backup

### 7.1 Setup Monitoring
```bash
# Enable Coolify monitoring
- Application metrics
- Database metrics
- System resources
- Error tracking
```

### 7.2 Configure Backups
```bash
# Database backup script
#!/bin/bash
BACKUP_DIR="/opt/coolify/backup/postgres-prod"
DATE=$(date +%Y%m%d_%H%M%S)
pg_dump -h workload-postgres-prod -U workload_user -d workload_production > "$BACKUP_DIR/backup_$DATE.sql"
gzip "$BACKUP_DIR/backup_$DATE.sql"
find "$BACKUP_DIR" -name "backup_*.sql.gz" -mtime +30 -delete
```

### 7.3 Setup Cron Jobs
```bash
# Add to crontab
0 2 * * * /opt/coolify/scripts/backup-database.sh
0 4 * * 0 /opt/coolify/scripts/cleanup-logs.sh
```

## Phase 8: GitHub Integration

### 8.1 Webhook Configuration
1. **GitHub Repository Settings**
   - Go to Settings → Webhooks
   - Add webhook URL: `https://coolify.example.com/webhooks/[webhook-id]`
   - Content type: `application/json`
   - Events: Push, Pull Request

2. **Coolify Webhook Setup**
   - Copy webhook URL from Coolify
   - Configure branch filters
   - Set deployment rules

### 8.2 GitHub Secrets
Add these secrets to GitHub repository:
```bash
COOLIFY_PRODUCTION_WEBHOOK=https://coolify.example.com/webhooks/prod-webhook-id
COOLIFY_STAGING_WEBHOOK=https://coolify.example.com/webhooks/staging-webhook-id
COOLIFY_TOKEN=your-coolify-api-token
```

## 🚨 Troubleshooting

### Common Issues

#### 1. Build Failures
```bash
# Check build logs
coolify logs --service=workload-app-prod --lines=100

# Common fixes
- Verify Dockerfile path
- Check environment variables
- Validate dependencies
```

#### 2. Database Connection Issues
```bash
# Test database connectivity
docker exec -it workload-postgres-prod psql -U workload_user -d workload_production

# Check network connectivity
docker network ls
docker network inspect coolify-network
```

#### 3. SSL Certificate Issues
```bash
# Regenerate certificates
coolify ssl renew --domain=workload.kemlu.go.id

# Check certificate status
openssl x509 -in /path/to/cert.pem -text -noout
```
