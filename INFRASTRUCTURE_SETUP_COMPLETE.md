## 🚀 Next Steps to Go Live

### Phase 1: GitHub Repository Setup (5 minutes)

1. **Create GitHub Repository**
   ```bash
   # Go to GitHub.com and create new repository:
   # Name: hpsb-workload-management
   # Visibility: Private (for government use)
   # Don't initialize with README
   ```

2. **Connect Local Repository**
   ```bash
   cd fullstack-dev
   git remote add origin https://github.com/YOUR_USERNAME/hpsb-workload-management.git
   git push -u origin main
   
   # Create staging branch
   git checkout -b staging
   git push -u origin staging
   
   # Return to main
   git checkout main
   ```

3. **Configure Repository Settings**
   - Enable branch protection for `main` and `staging`
   - Add required status checks
   - Require pull request reviews

### Phase 2: VPS Setup (30 minutes)

1. **Prepare Your VPS**
   ```bash
   # SSH into your VPS (Ubuntu 22.04+ recommended)
   ssh user@your-vps-ip
   
   # Download and run setup script
   curl -fsSL https://raw.githubusercontent.com/YOUR_USERNAME/hpsb-workload-management/main/scripts/vps-setup.sh | bash
   
   # Or run manually:
   git clone https://github.com/YOUR_USERNAME/hpsb-workload-management.git
   cd hpsb-workload-management
   chmod +x scripts/vps-setup.sh
   ./scripts/vps-setup.sh
   ```

2. **Access Coolify**
   ```
   URL: https://your-vps-ip:8080
   Complete initial setup wizard
   Create admin account
   ```

### Phase 3: Coolify Configuration (20 minutes)

1. **Create Project in Coolify**
   - Project Name: `HPSB Workload Management`
   - Description: `Production workload management system`

2. **Add GitHub Source**
   - Repository: `https://github.com/YOUR_USERNAME/hpsb-workload-management.git`
   - Add deploy key to GitHub repository

3. **Create Services**
   - PostgreSQL 15 (Production & Staging)
   - Redis 7 (Production & Staging)
   - Application (Production & Staging)

4. **Configure Environment Variables**
   ```bash
   # Production
   NODE_ENV=production
   DATABASE_URL=[Auto-generated by Coolify]
   JWT_SECRET=[Generate with: openssl rand -hex 32]
   ALLOWED_ORIGINS=https://workload.kemlu.go.id
   
   # Staging
   NODE_ENV=staging
   DATABASE_URL=[Auto-generated by Coolify]
   JWT_SECRET=[Different from production]
   ALLOWED_ORIGINS=https://staging.workload.kemlu.go.id
   ```

### Phase 4: GitHub Secrets Configuration (5 minutes)

Add these secrets to your GitHub repository:

```bash
# Go to Settings → Secrets and variables → Actions
COOLIFY_PRODUCTION_WEBHOOK=https://your-coolify.com/webhooks/prod-id
COOLIFY_STAGING_WEBHOOK=https://your-coolify.com/webhooks/staging-id
COOLIFY_TOKEN=your-coolify-api-token
```

### Phase 5: Domain Configuration (10 minutes)

1. **DNS Setup**
   ```
   # A Records pointing to your VPS IP:
   workload.kemlu.go.id → YOUR_VPS_IP
   staging.workload.kemlu.go.id → YOUR_VPS_IP
   ```

2. **SSL Certificates**
   - Coolify will automatically generate Let's Encrypt certificates
   - Verify SSL is working for both domains

### Phase 6: Deployment Testing (15 minutes)

1. **Test Staging Deployment**
   ```bash
   git checkout staging
   git push origin staging
   # Monitor deployment in Coolify
   # Verify: https://staging.workload.kemlu.go.id
   ```

2. **Test Production Deployment**
   ```bash
   git checkout main
   # Trigger manual deployment in Coolify UI
   # Verify: https://workload.kemlu.go.id
   ```
