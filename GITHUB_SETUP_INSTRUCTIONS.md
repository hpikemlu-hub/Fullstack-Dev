## 📖 Detailed Instructions

### Step 1: Create GitHub Repository

1. **Go to GitHub and create new repository**
   ```
   Repository name: hpsb-workload-management
   Description: HPSB Workload Management System - Production Enterprise Application
   Visibility: Private
   Initialize: Don't check any boxes (we have existing code)
   ```

2. **Connect your local repository**
   ```bash
   cd fullstack-dev
   git add .
   git commit -m "feat: Initial repository setup with complete infrastructure

   - Complete Next.js application with TypeScript
   - PostgreSQL database with Prisma ORM  
   - Authentication system with JWT
   - Employee and workload management modules
   - Calendar integration with real-time updates
   - Comprehensive testing suite
   - Docker containerization
   - CI/CD pipelines for staging and production
   - Coolify deployment configuration
   - VPS setup automation scripts
   - Security and monitoring configurations"
   
   git branch -M main
   git remote add origin https://github.com/YOUR_USERNAME/hpsb-workload-management.git
   git push -u origin main
   ```

### Step 2: Create Branch Structure

```bash
# Create and push staging branch
git checkout -b staging
git push -u origin staging

# Create and push development branch  
git checkout -b development
git push -u origin development

# Return to main
git checkout main
```

### Step 3: Configure Branch Protection

In GitHub repository settings:
1. Go to **Settings** → **Branches**
2. Add rule for `main` branch:
   - ✅ Require pull request reviews before merging
   - ✅ Require status checks to pass before merging
   - ✅ Require branches to be up to date before merging
   - ✅ Include administrators
3. Add rule for `staging` branch with similar settings

### Step 4: Add GitHub Secrets

Go to **Settings** → **Secrets and variables** → **Actions**

Add these repository secrets:
```bash
# Coolify Integration
COOLIFY_PRODUCTION_WEBHOOK=https://your-coolify-domain.com/webhooks/prod-webhook-id
COOLIFY_STAGING_WEBHOOK=https://your-coolify-domain.com/webhooks/staging-webhook-id  
COOLIFY_TOKEN=your-coolify-api-token

# Notifications (Optional)
SLACK_WEBHOOK=https://hooks.slack.com/services/YOUR/SLACK/WEBHOOK
EMAIL_WEBHOOK=https://api.example.com/send-email

# Container Registry
GHCR_TOKEN=${{ secrets.GITHUB_TOKEN }}  # Automatically available
```

### Step 5: VPS Preparation

**Option A: Automated Setup**
```bash
# SSH into your VPS
ssh user@your-vps-ip

# Download and run setup script
curl -fsSL https://raw.githubusercontent.com/YOUR_USERNAME/hpsb-workload-management/main/scripts/vps-setup.sh | bash
```

**Option B: Manual Setup**
```bash
# Clone repository on VPS
git clone https://github.com/YOUR_USERNAME/hpsb-workload-management.git
cd hpsb-workload-management

# Run setup script
chmod +x scripts/vps-setup.sh
./scripts/vps-setup.sh
```

### Step 6: Coolify Configuration

1. **Access Coolify**
   ```
   URL: https://your-vps-ip:8080
   ```

2. **Create Project**
   - Name: `HPSB Workload Management`
   - Description: `Production workload management system`

3. **Add GitHub Source**
   - Repository: `https://github.com/YOUR_USERNAME/hpsb-workload-management.git`
   - Deploy Key: Copy from Coolify and add to GitHub

4. **Create Services**
   - PostgreSQL 15 (Production & Staging)
   - Redis 7 (Production & Staging)
   - Application (Production & Staging)

### Step 7: Environment Configuration

**Production Environment Variables:**
```bash
NODE_ENV=production
DATABASE_URL=[Auto-generated by Coolify]
REDIS_URL=[Auto-generated by Coolify]
JWT_SECRET=[Generate with: openssl rand -hex 32]
JWT_REFRESH_SECRET=[Generate with: openssl rand -hex 32]
ALLOWED_ORIGINS=https://workload.kemlu.go.id
```

**Staging Environment Variables:**
```bash
NODE_ENV=staging
DATABASE_URL=[Auto-generated by Coolify]
REDIS_URL=[Auto-generated by Coolify]
JWT_SECRET=[Different from production]
JWT_REFRESH_SECRET=[Different from production]
ALLOWED_ORIGINS=https://staging.workload.kemlu.go.id
```

### Step 8: Test Deployments

```bash
# Test staging deployment
git checkout staging
git push origin staging
# Monitor deployment in Coolify

# Test production deployment
git checkout main
# Trigger manual deployment in Coolify UI
```
