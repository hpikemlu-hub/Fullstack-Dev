# 🚀 HPSB Workload Management System - Coolify Deployment Guide

This document provides complete instructions for deploying the HPSB Workload Management System to Coolify.

## 📋 Prerequisites

- VPS with Coolify installed and running
- Domain name pointing to your VPS IP address
- GitHub repository with the fullstack application code
- Basic understanding of Docker and environment variables

## 🚀 Quick Deployment to Coolify

### 1. Prepare Your VPS

1. Ensure Coolify is installed on your VPS
2. Make sure your domain points to your VPS IP address
3. Verify that ports 80, 443, and 3000 are accessible

### 2. Configure Your Coolify Instance

1. Log in to your Coolify dashboard
2. Create a new project (e.g., "workload-management")
3. Add your GitHub repository as a source

### 3. Set Up Database Services

1. **Add PostgreSQL Database**:
   - Name: `workload-postgres`
   - Version: `15`
   - Database: `workload_production`
   - Username: `workload_user`
   - Password: *Generate a secure password*

2. **(Optional) Add Redis Cache**:
   - Name: `workload-redis`
   - Version: `7`

### 4. Deploy the Application

1. **Add New Application** in Coolify:
   - Name: `workload-app`
   - Source: Select your GitHub repository
   - Build Pack: `Docker`
   - Dockerfile Path: `Dockerfile` (root directory)
   - Port: `3000`
   - Add the database dependency (workload-postgres)

2. **Configure Environment Variables**:
   ```bash
   # Required variables
   NODE_ENV=production
   DATABASE_URL=postgresql://workload_user:PASSWORD@workload-postgres:5432/workload_production
   JWT_SECRET=your_32_character_secret_here
   JWT_REFRESH_SECRET=your_32_character_refresh_secret_here
   ALLOWED_ORIGINS=https://yourdomain.com
   UPLOAD_PATH=/app/uploads
   NEXT_TELEMETRY_DISABLED=1
   LOG_LEVEL=info
   
   # Optional (if using Redis)
   REDIS_URL=redis://workload-redis:6379
   ```

3. **Generate JWT Secrets**:
   ```bash
   # Run on your local machine or any terminal
   openssl rand -hex 32  # For JWT_SECRET
   openssl rand -hex 32  # For JWT_REFRESH_SECRET
   ```

### 5. Configure SSL Certificate

1. In your application settings, go to the SSL tab
2. Enable Let's Encrypt for automatic SSL certificate generation
3. Verify your domain is properly configured

### 6. Start the Deployment

1. Click "Deploy" in the Coolify dashboard
2. Monitor the build logs to ensure successful deployment
3. Wait for the application to become healthy (check the health status)

## 🏗️ Architecture Overview

The application follows a microservices architecture with these components:

- **Frontend**: Next.js application with React components
- **Backend**: Next.js API routes with Prisma ORM
- **Database**: PostgreSQL for persistent data
- **Cache**: Redis for session and performance caching (optional)
- **File Storage**: Local filesystem (mounted volume) for file uploads

## 🔧 Configuration Details

### Dockerfile Configuration
- Uses multi-stage build for optimized image size
- Implements security best practices (non-root user)
- Includes health check endpoint verification
- Uses Node.js 18-alpine for compatibility

### Health Check
- Endpoint: `GET /api/health`
- Returns application and service health status
- Used by Coolify for container health monitoring

### Environment Variables
- `DATABASE_URL`: PostgreSQL connection string (auto-generated by Coolify when using database service)
- `JWT_SECRET`: Secret for JWT token signing
- `JWT_REFRESH_SECRET`: Secret for JWT refresh token signing
- `ALLOWED_ORIGINS`: CORS allowed origins
- `UPLOAD_PATH`: Path for file uploads
- `LOG_LEVEL`: Application logging level

## 🛠️ Troubleshooting

### Common Issues

1. **Build Fails**:
   - Check the build logs in Coolify dashboard
   - Ensure all dependencies are properly configured
   - Verify Node.js and npm versions match package.json

2. **Database Connection Issues**:
   - Verify DATABASE_URL is correctly set
   - Check that PostgreSQL service is running
   - Ensure the application can connect to the database service

3. **Health Check Fails**:
   - Verify the health endpoint `/api/health` is accessible
   - Check that all dependent services (database, Redis) are running

4. **SSL Certificate Issues**:
   - Ensure your domain DNS points to the VPS IP
   - Wait for DNS propagation if recently changed
   - Check Coolify SSL settings

### Debugging Steps

1. Check application logs in Coolify dashboard
2. Verify environment variables are set correctly
3. Test database connectivity from application container
4. Confirm health check endpoint returns 200 status
